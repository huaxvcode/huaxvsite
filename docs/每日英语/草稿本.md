# 草稿本

```
oj.cpp      处理和分析用户输入的命令
    
cps         多线程编译多个文件

equal       判断两个文本文件的内容在判题逻辑上是否相同

rn          运行单个目的程序文件，并以 in.txt 作为输入，run.txt 作为输出重定向，并判断程序文件的输入是否符合预期文件 out.txt 的内容

rno         运行单个目的程序文件，无任何重定向要求，也不用与预期文件 out.txt 的内容相比较

random      特殊运行用户编写的随机数据程序文件，该程序不接受任何的输入，但是输出需要重定向 in.txt 文件中

answer      特殊运行用户编写的预期程序文件，该程序以 in.txt 作为程序运行时的输入，并将该程序文件运行的输出结果重定向到 out.txt 文件中

oj_single   编译并且运行单个程序文件

oj          用随机数据程序文件的输出作为预期程序文件和目的程序文件的输入，将预期程序文件的输出与目的程序文件的输出做一次逻辑比较内容是否相同

oj_one      用随机数据程序文件的输出作为预期程序文件的输入，生成随机数据点和预期结果，生成的数据点可能只有一个，分别重定向到 in.txt 和 out.txt 文件中

oj_get      用随机数据程序文件的输出作为预期程序文件的输入，生成随机数据点和预期结果，生成的数据点可能有多个，都从 1 开始编号，存放到 datas 文件夹中

oj_get_run  用 datas 文件夹中的所有数据点对目的程序文件进行测试，如果测试通过输出 accept 状态，否则输出 error 状态，输出结果保存到文件夹 runs 中


deal_file   切割出「文件名」和「文件后缀名」

is_right    判断「文件名」和「文件后缀名」是否合适

main        主程序
```
